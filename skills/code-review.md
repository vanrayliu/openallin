# Code Review — 代码审查

> 在完成任务、实现主要功能或合并前使用此技能，验证工作是否符合要求。

## 审查维度

### 1. 正确性
- 实现是否符合规格/需求？
- 是否有逻辑错误？
- 边界条件是否处理？

### 2. 完整性
- 所有任务项是否完成？
- 是否有遗漏的功能点？
- 测试是否覆盖关键路径？

### 3. 一致性
- 代码风格是否与项目一致？
- 命名是否清晰？
- 架构模式是否遵循？

### 4. 安全性
- 是否有安全隐患？
- 输入是否验证？
- 敏感信息是否泄露？

### 5. 可维护性
- 代码是否可读？
- 是否有不必要的复杂度？
- 依赖是否合理？

## 审查流程

### Step 1: 理解变更
- 阅读变更摘要
- 理解变更目的
- 查看相关文件

### Step 2: 逐行审查
- 关注逻辑而非格式
- 标记问题点
- 提出改进建议

### Step 3: 验证测试
- 测试是否充分？
- 是否覆盖关键路径？
- 是否有边缘情况未覆盖？

### Step 4: 给出结论
- 通过 / 需要修改 / 拒绝
- 列出必须修改项
- 列出建议修改项

## 反馈原则

- 具体而非模糊
- 建设性而非批评性
- 基于事实而非偏好
- 区分"必须改"和"建议改"
