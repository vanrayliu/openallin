# /oa-discuss — 变更提案需求讨论

当用户输入 `/oa-discuss` 时执行此技能。

## 用途区分

**oa-discuss vs oa-brainstorming**：

| 命令 | 用途 | 输出 |
|------|------|------|
| `/oa-discuss` | 讨论**已有变更提案**的需求细节 | 完善 proposal.md |
| `/oa-brainstorming` | 探索**新功能/新想法**，开放式创意讨论 | 创建新的 change proposal |

**何时使用 oa-discuss**：
- 已经有 `changes/<name>/proposal.md`
- 需要细化需求、澄清边界条件
- Spec-Driven Workflow 的讨论阶段

**何时使用 oa-brainstorming**：
- 想要创建新功能，但不知道具体需求
- 探索性讨论，开放式问题
- 需要创意发散，多方案对比

## 核心设计：多轮迭代

```
┌─────────────────────────────────────────────────────────────┐
│  /oa-discuss 是多轮讨论过程，每轮包含：                      │
│                                                             │
│  1. AI 提出问题 → 2. 用户回答 → 3. 总结确认 → 4. 下一轮    │
│                                                             │
│  最多 5 轮，或直到用户确认讨论充分                           │
└─────────────────────────────────────────────────────────────┘
```

## 工作流程

### 1. 读取变更提案
打开 `changes/<name>/proposal.md`，了解当前提案内容。

### 轮次结构

```
第 N 轮
━━━━━━━━
1. 聚焦主题（这轮要讨论什么）
2. Q&A 讨论
3. 更新 proposal.md
4. 用户决策
```

### 第 1 轮：聚焦意图
```
问题示例：
- 你想解决什么问题？
- 期望的结果是什么样的？
- 为什么现在需要这个？
```

### 第 2 轮：聚焦边界
```
问题示例：
- 输入是什么？输出是什么？
- 边界情况有哪些？
- 什么算成功？什么算失败？
```

### 第 3 轮：聚焦环境
```
问题示例：
- 与现有系统的交互点在哪里？
- 技术限制是什么？
- 性能、安全、兼容性要求？
```

### 第 4 轮：聚焦方案
```
- 提出 2-3 种实现方案
- 分析各方案优缺点
- 推荐最适合的方案
```

### 第 5 轮：聚焦确认
```
- 明确范围（做什么 / 不做什么）
- 确认技术选型
- 制定验收标准
```

## 轮次总结格式

每轮结束时输出：

```markdown
## 第 N 轮总结

### ✅ 已确认
- ...

### ❓ 待确认
- ...

### 用户决策
[ ] 继续下一轮
[ ] 讨论充分，进入 /oa-validate
[ ] 需要修改前面的内容
```

## 状态保存

每轮结束后更新 `changes/<name>/proposal.md`：

```markdown
# 变更提案: <name>

> 创建时间: YYYY-MM-DD
> 更新时间: YYYY-MM-DD
> 轮次: N/5
> 状态: 讨论中/已确认

## 概述
[简要描述此变更的目的和动机]

## 需求分析（已确认）
### 用户故事
- 作为 [角色]，我希望 [功能]，以便 [收益]

## 边界条件（已确认）
- 输入：...
- 输出：...
- 成功定义：...
- 失败定义：...

## 技术选型（已确认）
- 方案：...
- 理由：...

## 验收标准（已确认）
- [ ]

## 影响范围
- 

## 待确认问题
- ...
```

## 终止条件

1. 用户说"够了"、"可以了"、"进入验证"
2. 完成 5 轮讨论
3. 用户明确决策

## 最终输出

讨论结束后，`changes/<name>/proposal.md` 包含：

1. **核心意图** — 用户真正想要什么
2. **边界条件** — 成功/失败定义
3. **技术选型** — 为什么用这个方案
4. **验收标准** — 怎么算完成
5. **下一步** — `/oa-validate` 或 `/oa-apply`

## 相关 Skills

- `/oa-propose` — 创建变更提案（前置）
- `/oa-validate` — 验证规格文档格式（后置）
- `/oa-apply` — 执行变更任务清单（后置）
- `/oa-brainstorming` — 探索性创意讨论（不同用途）
