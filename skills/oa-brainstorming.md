# /oa-brainstorming — 头脑风暴与需求探索

当用户输入 `/oa-brainstorming <topic>` 时执行此技能。

## 核心设计：多轮迭代

```
┌─────────────────────────────────────────────────────────────┐
│  /oa-brainstorming 是多轮讨论过程，每轮包含：                │
│                                                             │
│  1. AI 提出问题 → 2. 用户回答 → 3. 总结确认 → 4. 下一轮  │
│                                                             │
│  最多 5 轮，或直到用户确认讨论充分                           │
└─────────────────────────────────────────────────────────────┘
```

## 工作流程

### 轮次结构

```
第 N 轮
━━━━━━━━
1. 聚焦主题（这轮要讨论什么）
2. Q&A 讨论
3. 轮次总结
4. 用户决策
```

### 第 1 轮：聚焦意图
```
问题示例：
- 你想解决什么问题？
- 期望的结果是什么样的？
- 为什么现在需要这个？
```

### 第 2 轮：聚焦边界
```
问题示例：
- 输入是什么？输出是什么？
- 边界情况有哪些？
- 什么算成功？什么算失败？
```

### 第 3 轮：聚焦环境
```
问题示例：
- 与现有系统的交互点在哪里？
- 技术限制是什么？
- 性能、安全、兼容性要求？
```

### 第 4 轮：聚焦方案
```
- 提出 2-3 种实现方案
- 分析各方案优缺点
- 推荐最适合的方案
```

### 第 5 轮：聚焦确认
```
- 明确范围（做什么 / 不做什么）
- 确认技术选型
- 制定验收标准
```

## 轮次总结格式

每轮结束时输出：

```markdown
## 第 N 轮总结

### ✅ 已确认
- ...

### ❓ 待确认
- ...

### 用户决策
[ ] 继续下一轮
[ ] 讨论充分，进入 /oa-plan
[ ] 需要修改前面的内容
```

## 状态保存

每轮结束后保存到 `workspace/brainstorm-YYYY-MM-DD.md`：

```markdown
# 头脑风暴记录

> 日期: YYYY-MM-DD
> 主题: <用户需求>
> 轮次: N/5
> 状态: 进行中/已完成

## 讨论历史

### 第 1 轮：...
### 第 2 轮：...

## 已确认
- 核心意图: ...
- 边界条件: ...

## 最终输出
（讨论结束后填写）
```

讨论结束后，同时创建 `workspace/CONTEXT.md` 供 `/oa-plan` 使用。

## 终止条件

1. 用户说"够了"、"可以了"、"进入规划"
2. 完成 5 轮讨论
3. 用户明确决策

## 最终输出

讨论结束后，`workspace/CONTEXT.md` 包含：

1. **核心意图** — 用户真正想要什么
2. **边界条件** — 成功/失败定义
3. **技术选型** — 为什么用这个方案
4. **验收标准** — 怎么算完成
5. **下一步** — `/oa-plan` 或 `/oa-propose`
