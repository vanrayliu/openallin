# /oa-brainstorming — 探索性创意讨论

当用户输入 `/oa-brainstorming <topic>` 时执行此技能。

## 用途区分

**oa-brainstorming vs oa-discuss**：

| 命令 | 用途 | 输出 |
|------|------|------|
| `/oa-brainstorming` | 探索**新功能/新想法**，开放式创意讨论 | 创建新的 change proposal |
| `/oa-discuss` | 讨论**已有变更提案**的需求细节 | 完善 proposal.md |

**何时使用 oa-brainstorming**：
- 想要创建新功能，但不知道具体需求
- 探索性讨论，开放式问题
- 需要创意发散，多方案对比
- "我想做一个 X，但不知道怎么开始"

**何时使用 oa-discuss**：
- 已经有 `changes/<name>/proposal.md`
- 需要细化需求、澄清边界条件
- Spec-Driven Workflow 的讨论阶段

## 核心设计：发散→收敛

```
┌─────────────────────────────────────────────────────────────┐
│  /oa-brainstorming 是创意发散→收敛过程：                     │
│                                                             │
│  发散（探索可能性）→ 收敛（选择方案）→ 确认（形成提案）     │
│                                                             │
│  不限制轮次，直到用户说"确定这个方案"                        │
└─────────────────────────────────────────────────────────────┘
```

## 工作流程

### 发散阶段：探索可能性

```
目标：尽可能多地探索可能性

1. 理解主题
   - 用户想做什么？
   - 核心问题是什么？
   - 有什么约束？

2. 发散提问
   - 有没有类似的解决方案？
   - 可以从哪些角度切入？
   - 技术上有什么可能性？
```

### 收敛阶段：选择方案

```
目标：从多个可能性中选择最适合的方案

1. 方案对比
   - 列出 2-5 种可行方案
   - 分析各方案优缺点
   - 评估复杂度、风险、收益

2. 推荐方案
   - 根据用户需求推荐最适合的方案
   - 说明推荐理由
```

### 确认阶段：形成提案

```
目标：将选定方案转化为具体提案

1. 生成提案
   - 创建 `changes/<topic>/proposal.md`
   - 包含概述、用户故事、验收标准

2. 用户确认
   - 提案是否符合预期？
   - 是否需要调整？
```

## 输出格式

### 发散阶段输出

```markdown
## 💡 创意发散

### 方案 A: [名称]
- 描述: ...
- 优点: ...
- 缺点: ...
- 适用场景: ...

### 方案 B: [名称]
- 描述: ...
- 优点: ...
- 缺点: ...
- 适用场景: ...

### 方案 C: [名称]
- 描述: ...
- 优点: ...
- 缺点: ...
- 适用场景: ...

### 其他可能性
- ...
```

### 收敛阶段输出

```markdown
## 🎯 方案收敛

### 推荐方案: [名称]

**理由**:
- 与用户需求匹配度高
- 实现复杂度适中
- 风险可控

**实施路径**:
1. 第一步: ...
2. 第二步: ...
3. 第三步: ...

**预期效果**:
- ...

### 是否确认这个方案？
[ ] 确认，进入提案创建
[ ] 我想选择其他方案
[ ] 继续探索更多可能性
```

### 确认阶段输出

创建 `changes/<topic>/proposal.md`：

```markdown
# 变更提案: <topic>

> 创建时间: YYYY-MM-DD
> 来源: brainstorming 讨论
> 选定方案: [方案名称]

## 概述
[基于 brainstorming 确定的内容]

## 需求分析
### 用户故事
- 作为 [角色]，我希望 [功能]，以便 [收益]

## 验收标准
- [ ]

## 影响范围
- 

## 技术方案
- 方案: [选定方案]
- 理由: ...
```

## 状态保存

讨论过程中保存到 `workspace/brainstorm-YYYY-MM-DD.md`：

```markdown
# Brainstorming 记录

> 日期: YYYY-MM-DD
> 主题: <用户需求>
> 状态: 发散中/收敛中/已确认

## 讨论历史

### 发散阶段
- 方案 A: ...
- 方案 B: ...
- 方案 C: ...

### 收敛阶段
- 推荐方案: ...

## 最终方案
（确认后填写）
```

## 终止条件

1. 用户确认方案："就用这个方案"
2. 用户说"够了，创建提案"
3. 用户取消："暂时不做了"

## 最终输出

Brainstorming 结束后，创建 `changes/<topic>/` 目录结构：

```
changes/<topic>/
├── proposal.md    # 提案文档
├── design.md      # 设计文档（可选）
└── tasks.md       # 任务清单（可选）
```

## 相关 Skills

- `/oa-propose` — 直接创建变更提案（不经过 brainstorming）
- `/oa-discuss` — 讨论已有提案的需求细节（不同用途）
- `/oa-ui-design` — UI/UX 设计智能系统（可结合使用）
- `/oa-writing-plans` — 计划编写（后置）
