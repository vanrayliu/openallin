# Verification Loop — 验证闭环

> 在声称工作完成、修复或测试通过之前使用此技能。

## 核心原则

1. **证据优先** — 在做出任何成功声明之前运行验证命令
2. **实际验证** — 不要假设，运行命令并确认输出
3. **多维度** — 从多个角度验证（lint、test、typecheck、manual）
4. **可回溯** — 验证结果必须记录在案

## 验证维度

### 1. 完整性验证
- [ ] 所有任务项已完成？
- [ ] 所有规格要求已实现？
- [ ] 没有遗漏的功能点？

### 2. 正确性验证
- [ ] 实现符合规格意图？
- [ ] 没有偷工减料？
- [ ] 没有绕过校验？

### 3. 一致性验证
- [ ] 代码与架构决策一致？
- [ ] 风格与项目一致？
- [ ] 没有引入不必要的依赖？

### 4. 质量验证
- [ ] Lint 通过？
- [ ] 类型检查通过？
- [ ] 测试全部通过？
- [ ] 构建成功？

## 工作流

### Step 1: 定义验证标准
- 从规格中提取验证条件
- 确定需要运行的命令

### Step 2: 执行验证
```bash
# 根据项目实际使用的工具运行验证命令
# 示例（Node.js 项目）:
npm run lint
npm run typecheck
npm test
npm run build

# 示例（Python 项目）:
ruff check .
mypy .
pytest
```

### Step 3: 分析结果
- 记录通过/失败
- 分析失败原因
- 确定是否需要修复

### Step 4: 修复并重新验证
- 如果有任何失败，修复后重新验证
- 直到所有验证通过

### Step 5: 记录结果
- 保存验证报告
- 更新任务状态

## 输出物

- 验证报告
- 测试覆盖率报告
- 构建状态
