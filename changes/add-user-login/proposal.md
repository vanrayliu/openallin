# 提案: 用户登录系统

## 为什么
项目需要用户认证能力，支持邮箱注册、登录、会话管理，为后续功能提供身份基础。

## 范围
### 包含
- 邮箱注册流程（含验证邮件）
- 邮箱+密码登录
- JWT Token 颁发与刷新
- 会话超时管理
- 密码重置流程

### 不包含
- 第三方 OAuth 登录（后续迭代）
- 多因素认证（后续迭代）
- 用户权限/角色系统（后续迭代）

## 影响
- 新增 auth 模块
- 新增用户表
- 新增邮件服务集成
- 所有 API 端点需要认证中间件

## 成功标准
- 用户可以完成注册→验证→登录全流程
- Token 刷新机制正常工作
- 会话超时后正确拒绝请求
- 所有安全规则通过审查
