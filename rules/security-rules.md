# 安全规则

## 密钥管理

1. 永远不要提交密钥、密码、Token
2. 使用环境变量管理敏感信息
3. 使用 .gitignore 排除敏感文件
4. 定期轮换密钥

## 输入验证

1. 所有外部输入必须验证
2. 使用白名单而非黑名单
3. 验证类型、长度、格式、范围
4. 对 SQL、HTML、命令注入保持警惕

## 认证授权

1. 使用成熟的认证库
2. Token 设置合理的过期时间
3. 实施最小权限原则
4. 敏感操作需要二次确认

## 数据传输

1. 使用 HTTPS
2. 敏感数据加密传输
3. 不要在 URL 中传递敏感信息
4. 设置安全的 Cookie 属性

## 依赖安全

1. 定期更新依赖
2. 使用锁文件
3. 扫描已知漏洞
4. 审计新依赖
